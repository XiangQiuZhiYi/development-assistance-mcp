# 🚨 重要：每次对话必须执行的检查清单

## ⚠️ 第一步：立即读取开发指南（强制要求）

**在生成任何代码或提供建议之前，必须按以下顺序读取文件：**

### 1️⃣ 必读：`.vscode/CORE_GUIDELINES.md` 🆕

```
⭐⭐⭐ 最高优先级 - 强制性规范，绝不可违反
🚫 包含禁止事项和核心约定
🔒 所有代码生成必须严格遵循此文件中的强制要求
```

**关键内容：**
- 禁止使用的技术或写法（如：禁用 .vue 文件，完全 TSX 模式）
- 强制性命名规范和代码约定
- 核心开发规范（如：弹框组件必须使用 handleOpen 模式）
- 安全规范和限制

**如果不存在**：使用 `mcp_my-mcp-server_read_core_guidelines` 读取，如返回不存在则跳过

### 2️⃣ 必读：`.vscode/PROJECT_GUIDE.md`

```
⭐⭐ 项目级规范
🏗️ 包含项目结构、技术栈、命名规范
📚 提供实际用例和代码示例
```

**关键内容：**
- 框架和技术栈信息
- 路由系统规范
- 状态管理方案
- 实际开发用例和最佳实践

**如果不存在**：使用 `mcp_my-mcp-server_read_project_guide` 读取

### 3️⃣ 参考：`.vscode/PERSONAL_DEV_NOTES.md`

```
⭐ 个人代码片段库
💾 包含开发者收集的优质组件、工具函数、Hooks

```

**关键内容：**
- 优质组件和工具函数
- 开发技巧和经验总结
- 可跨项目复用的代码片段

**如果不存在**：使用 `mcp_my-mcp-server_read_personal_notes` 读取

## 💡 开发指南应用原则

**在解决问题或生成代码时：**

1. **首先严格遵守 CORE_GUIDELINES.md（如果存在）**
   - 这是强制性规范，绝对不可违反
   - 检查是否有禁止使用的技术或写法
   - 例如：如果禁用 .vue 文件，则必须使用 TSX
   - 例如：如果禁用 v-model 控制弹框，则必须使用 handleOpen 模式
   - 任何违反核心必读的代码都是不可接受的

2. **其次参考 PROJECT_GUIDE.md**
   - 遵循项目的命名规范、文件组织结构
   - 使用项目中已有的组件和工具函数
   - 遵循双语规范、样式规范等
   - 生成代码时优先查看实际用例里是否有类似场景，复用其中的模式
   - 如果没有类似用例，再去其他模块寻找参考

3. **最后参考 PERSONAL_DEV_NOTES.md**
   - 查找是否有可复用的组件、函数或 Hooks
   - 使用开发者记录的代码片段和技巧
   - 优先使用已验证的优质代码，避免重复造轮子

4. **遇到指南中提到的场景**
   -核心必读相关 🆕
